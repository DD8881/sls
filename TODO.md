## Оновлення даних (поточний ручний процес)

Щоб підтягнути свіжі знижки і викотити на прод, зараз треба запустити вручну:

```bash
# 1. Скрап усіх мереж/міст у discounts.db (8 мереж, ~10 паралельних воркерів)
python3 run_scraper.py
#    опційно вужче:  --city Київ   --chain silpo   --workers 5

# 2. Перегенерувати статику й задеплоїти Worker
#    (deploy.sh всередині: generate_static.py -> зібрати ./public -> wrangler deploy)
./deploy.sh
```

Тобто весь pull нових даних = **дві команди**: `run_scraper.py` → `deploy.sh`.
SQLite (`discounts.db`) живе лише локально як build-time проміжний шар; на хостингу
БД немає (див. DEPLOY.md).

## [ ] Автоматизувати локально через launchd (ще не реалізовано)

Ідея: одна `launchd`-джоба раз на день ганяє обидві команди без участі людини.

- `launchd` plist із `StartCalendarInterval` (напр. щодня о 06:00).
- Обгортка-скрипт: `caffeinate -i` на час запуску, щоб Mac не заснув посеред скрапу;
  якщо ноут спав у призначений час — джоба має відпрацювати при пробудженні.
- Скрипт = `python3 run_scraper.py && ./deploy.sh`, з логом у файл і нотифікацією
  про фейл (скрап може впасти на Cloudflare-челенджі ATB чи rate-limit).
- Альтернатива «все в одному»: Oracle Cloud Always Free VM з cron — якщо не хочеться
  залежати від того, чи увімкнений Mac.

> Реалізовувати поки не треба — це просто нотатка на майбутнє.
